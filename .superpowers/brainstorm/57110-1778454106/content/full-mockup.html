Think of an embedding as a coordinate for meaning. You take a word, sentence, or document and run it through a model that produces a fixed-size float array — say, 1536 dimensions for OpenAI's text-embedding-3-small.

Two sentences that mean similar things will produce vectors that are close together in that space — measurable with cosine similarity. That's the core idea.
# distance between two embeddings
import numpy as np

similarity = np.dot(vec_a, vec_b) / (
  np.linalg.norm(vec_a) * np.linalg.norm(vec_b)
) # → 0.0 to 1.0
