What to ship (final design intent — distilled from chat2.md)

  • Palette: restrained. Tabby (primary #ff7a1a) + Grape (secondary) only. No rainbow tiles. Mint = quiet success dot. Maple = overdue cards only. Teal/sky/lemon/blush dropped from main surface.
  • Typography: Space Grotesk (display) + Space Mono (body)
  • Layout: Three-pane — left rail (nav/identity), center chat, right sidebar (calendar at top, restrained cards below)
  • Chat: No bubbles. Line-per-message, minimal chrome. Small mascot peek only on Kitty's FIRST reply.
  • Calendar widget: Feb 2026 grid at top-right, today highlighted in tabby, dots under days with events
  • Mascot: Gentle 4-second bob animation on rail mark + empty-state hero + status pet
  • Highlighted items: Background color with opacity (not full saturation) — see the calendar today-marker pattern

Anti-patterns (things the user explicitly rejected)

  • Chat bubbles ("clutters up the whole interaction, very unnecessary")
  • Rainbow tile colors on briefs/cards
  • Avatars + labels on every message
  • Lemon as a primary star (demoted to spice-only)

Where it lands in the kitty-chat codebase

  • Target: gateway/kitty-chat/ (Next.js 16.2.6, Turbopack, port 4000)
  • The kitty-rail prototype uses CDN React 18 + Babel inline — do NOT copy that structure. Reimplement as Next.js components.
  • Tokens first: port colors_and_type.css into kitty-chat/app/globals.css or a Tailwind theme — every component depends on these vars.
  • SVG assets: copy from design-system/project/assets/ → kitty-chat/public/. Use Next's <Image> or inline SVG depending on animation needs.
